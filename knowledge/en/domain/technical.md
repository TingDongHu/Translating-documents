## General Principles

- Use precise technical terminology consistent with industry standards (ISO, IEC, etc.)
- Preserve all measurements, tolerances, and specifications exactly
- Use SI units; convert only if explicitly required
- Maintain consistent terminology throughout the document
- Keep model numbers, part numbers, and serial numbers in original form

## Technical Writing Style

- Use imperative mood for instructions ("Press the button" not "You should press the button")
- Use passive voice for specifications ("The valve is operated by...")
- Be concise — avoid unnecessary words
- Use numbered lists for sequential steps

## Units and Measurements

- Preserve original units and their exact values
- Common: mm, cm, m, kg, g, L, mL, °C, °F, s, min, h, V, W, bar, Pa, psi
- Temperature: preserve Celsius by default; add Fahrenheit in parentheses if needed
- Do NOT convert between unit systems unless explicitly instructed

## Standards and Certifications

- Preserve standard numbers exactly (ISO 9001, CE marking, GB/T 12345, etc.)
- Translate certification body names to English equivalents where established
- Keep standard acronyms (CE, UL, FCC, CCC, RoHS, REACH, WEEE, etc.)

## User Manuals and Instructions

- Translate section headings and control/indicator labels consistently
- Safety warnings (危险/警告/注意) → map to "DANGER" / "Warning" / "Caution" consistently
- Cross-references to other sections: preserve page references, translate section names
- "请参阅" → "See" / "Please refer to"
- "注：" → "Note:"
- Numbers in bullet lists: keep original numbering (01, 02, etc.)

## Product Documentation

- Spec sheets: tables with parameter name + value pairs, keep table structure
- Exploded diagrams: part labels translate, number references keep as-is
- Troubleshooting tables: "现象" → "Symptom", "原因" → "Cause", "处理方法" → "Solution"
- Packing lists: item names translate, quantities preserve
