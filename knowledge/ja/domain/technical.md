# 技術文書 (Technical Documents) — ja-JP

## Reader Model

**読者像：** 技術者、研究開発者、エンジニア（機械・電気・化学・土木・情報工学等）、特許調査者、規格認証担当者、安全管理者、製品責任者。母語話者かつ専門知識保有者で、原文の技術的厳密性を完全に保持した翻訳を要求する。以下の点に特に敏感である：

- **用語の厳密性**：JIS規格で定義された用語と一般用語の差異を即座に検出する。「精度」と「確度」、「許容差」と「公差」の混同を見逃さない。
- **数値・単位の正確性**：単位記号の誤記（kWとKW、mmとMM等）や数値の誤変換を絶対に許容しない。SI接頭辞（k, M, G, m, μ, n等）の表記誤りは文書全体の信頼を失墜させる。
- **安全情報の完全性**：警告文・注意文の強度レベル（危険/警告/注意）の誤訳は、製品事故や法的責任に直結するため、一切の誤差を許容しない。
- **規格引用の正確性**：JIS、ISO、IEC、ANSI等の規格番号・発行年の引用誤りは、技術的根拠の崩壊につながる。
- **図表・参照の一貫性**：「図3を参照」と訳した箇所と実際の図番号が一致しない等の基本的な誤りを厳しく指摘する。
- **手順記述の明確性**：操作手順の順序誤り、条件分岐の不明確さ、安全注意の配置ミスを検出する。
- **業界固有の用法**：半導体、自動車、化学プラント等、業界ごとに確立された用語体系（例：半導体の「ウェハ」対「ウエハ」）の表記ゆれを問題視する。

翻訳結果が原文の技術的正確性を完全に保持しているかを最優先で評価し、日本語としての流暢さよりも正確性を重視する読者層である。ただし、読みにくさが誤解を生むレベルに達してはならない。

## Decision Framework

### JIS定義と一般用語の優先順位
| Condition | Decision | Example |
|-----------|----------|---------|
| 専門用語にJIS規格の定義が存在する | JIS定義を正規訳として採用する。一般用語とJIS定義が異なる場合もJISを優先 | "tolerance" → 「公差」（JIS B 0401）、一般的な「許容誤差」は避ける |
| 同一用語に複数のJIS定義がある | 文書の該当分野の専門JISを優先。分野間で定義が競合する場合は出典を注記 | "stress" → 機械分野では「応力」、言語学分野では「強勢」 |
| JIS未定義の専門用語 | 業界団体規格（JEITA、JASO、JSWAS等）またはISO定義を参照 | "Ethernet" → 「イーサネット」（IEEE 802.3準拠） |
| 規格番号の引用 | 番号・発行年を原文ママ保持。ハイフン位置・大文字小文字も含め一切変更不可 | "ISO 9001:2015" → 「ISO 9001:2015」（×「ISO9001:2015」、×「ISO 9001-2015」） |
| 部品番号・型式番号 | 数字・記号・区切り文字を含め完全原文維持 | "M6×20 SUS304" → 「M6×20 SUS304」 |

### SI単位と尺貫法・ヤードポンド法の変換方針
| Condition | Decision | Example |
|-----------|----------|---------|
| 原文がSI単位系 | そのままSI単位で表記。単位記号はJIS Z 8203に従う | "10 mm" → 「10 mm」（「10ミリ」は不可） |
| 原文がヤードポンド法（inch, foot, psi等） | 原則、括弧内にSI換算値を併記。技術分野では原単位保持が基本 | "1 inch" → 「1インチ（25.4 mm）」 |
| 原文が日本の尺貫法（寸、尺、間等） | 現代技術文書では原則SI換算。歴史的文献では尺貫法を保持し換算値併記 | "1尺" → 「1尺（約303 mm）」 |
| SI接頭辞の表記 | JIS Z 8203に従い、k（キロ）、M（メガ）、G（ギガ）、m（ミリ）、μ（マイクロ）、n（ナノ）を正しく表記 | "kW"（正）、"KW"（誤）、"MHz"（正）、"MHZ"（誤） |
| 単位記号の複数形不使用 | 日本語では単位記号に複数形はない。英語の複数形"gs"、"kgs"等を単数形に修正 | "10 kg"（正、×"10 kgs"）、"5 mm"（正、×"5 mms"） |
| 単位の前後スペース | 数値と単位の間は半角スペース1つ（JIS Z 8301推奨）。%と℃は数値に密着も許容 | "10 mm"（正）、"10mm"（分野により許容） |

### 安全警告レベルの訳分け（危険/警告/注意/重要）
| Condition | Decision | Example |
|-----------|----------|---------|
| "DANGER" | 「危険」— 死亡または重傷に直結する切迫した危険状態 | "DANGER: High Voltage" → 「危険：高電圧」 |
| "WARNING" | 「警告」— 死亡または重傷につながる可能性がある危険状態 | "WARNING: Hot Surface" → 「警告：高温部」 |
| "CAUTION" | 「注意」— 軽傷または物的損害につながる可能性がある状態 | "CAUTION: Pinch Hazard" → 「注意：挟まれ注意」 |
| "NOTICE" / "IMPORTANT" | 「重要」— 物的損害やシステム障害を防ぐための注意喚起 | "IMPORTANT: Back up data" → 「重要：データをバックアップしてください」 |
| 安全警告の階層構造 | 原文の階層順序を保持。階層の逆転（危険よりも警告を上位に配置等）は絶対に不可 | DANGER > WARNING > CAUTION > NOTICE の順序を保持 |
| 安全シンボル（△!等） | ISO 7010またはJIS Z 9101の安全標識に従い、該当するピクトグラムの説明を併記 | "△!" → 「警告マーク」 |
| SAFETY INSTRUCTIONS | 「安全上のご注意」— 冒頭の総合安全章の固定訳 | "SAFETY INSTRUCTIONS" → 「安全上のご注意」 |

### カタカナ語と和語・漢語の選択
| Condition | Decision | Example |
|-----------|----------|---------|
| 日本語に対応概念がある既存用語 | 和語・漢語を優先。不必要なカタカナ語は避ける | "implementation" → 「実装」（×インプリメンテーション）、"configuration" → 「構成」（×コンフィギュレーション） |
| 業界でカタカナ表記が標準化 | 業界標準のカタカナ表記に従う。ただし長音表記（ー）の有無を業界内で統一 | "computer" → 「コンピュータ」（JIS）、"processor" → 「プロセッサ」 |
| 日本語に対応概念がない | カタカナ語を使用し、初出時に原語または説明を併記 | "stakeholder" → 「ステークホルダー（利害関係者）」 |
| 原文の特定概念を強調する必要がある場合 | カタカナ語を使用し、必要に応じて和語を併記 | "design review" → 「デザインレビュー（設計審査）」 |
| 英語略語（CPU, API, HTTP等） | 日本業界で通用する略語はそのまま使用。漢語訳は不要 | "API" → 「API」（×「アプリケーションプログラミングインタフェース」と毎回訳さない） |

### モダリティと義務表現（shall/must/should/may/can）
| Condition | Decision | Example |
|-----------|----------|---------|
| "shall"（規格・契約上の義務） | 「…するものとする」JIS文書での標準訳。法律文書でも同様 | "The device shall comply with IEC 61000." → 「本装置はIEC 61000に適合するものとする。」 |
| "must"（強い義務・必須条件） | 「…しなければならない」実際上の強制を表す | "The operator must wear safety goggles." → 「操作員は安全ゴーグルを着用しなければならない。」 |
| "should"（推奨・望ましい行為） | 「…することが望ましい」「…するのがよい」強制ではない推奨 | "The cable should be shielded." → 「ケーブルはシールドすることが望ましい。」 |
| "may"（許可・許容） | 「…することができる」禁止されていない行為 | "The user may adjust the gain setting." → 「使用者はゲイン設定を調整することができる。」 |
| "can"（可能性・能力） | 「…できる」「…することが可能である」技術的可能性 | "The system can process 1000 transactions per second." → 「本システムは毎秒1,000トランザクションを処理できる。」 |
| "need not"（不必須） | 「…する必要はない」義務の不存在 | "The device need not be grounded." → 「本装置は接地する必要はない。」 |

### 受動態と能動態の選択
| Condition | Decision | Example |
|-----------|----------|---------|
| 操作手順の説明 | 能動態を基本。動作主（ユーザ）を主語にした能動文 | "Press the START button." → 「STARTボタンを押す。」（×「STARTボタンが押される。」） |
| 事実・結果の記述 | 受動態も許容。無主文も使用可 | "The temperature was measured at 85°C." → 「温度は85°Cで測定された。」 |
| 動作主が不明・不重要な場合 | 無主文または受動態を使用。動作主を特定しない | "A new method was developed." → 「新たな手法が開発された。」 |
| 英語のby句を伴う受動態 | 動作主が重要な場合のみ残し、それ以外は省略または能動態に変換 | "The sample was analyzed by HPLC." → 「試料をHPLCで分析した。」 |
| 日本語の「られる」受動の回避 | 「…されている」形式は状態表現として許容。被害受動にならないよう注意 | "The valve is closed." → 「バルブは閉じている。」（×「閉じられている」だが状態として許容） |

### 数値・単位の完全一致（トレラントゼロ）
| Condition | Decision | Example |
|-----------|----------|---------|
| 原文の数値 | 翻訳後も完全に一致。四捨五入・切捨て・桁区切り変更は一切不可 | "10,000.5" → 「10,000.5」（×「10000.5」、×「10,000.50」） |
| 小数点記号 | 日本語はピリオド（.）を使用。コンマ（,）の原文は注意深く変換 | "3,141.59" → 「3,141.59」（英語スタイル）、"3.141,59" → 「3,141.59」（欧州スタイル→日本スタイルに変換） |
| 桁区切り | 3桁ごとにカンマ。日本語では補助的にスペースも許容 | "1,234,567" → 「1,234,567」 |
| 単位記号 | JIS Z 8203準拠の国際単位記号を正しく使用 | "mm"（正）、"Millimeter"（文章中以外は不可）、"ミリメートル"（文章中では許容） |
| 割合・百分率 | "%"を基本。文章中では「パーセント」も可。1文書内で統一 | "5%" → 「5%」 |
| 温度表示 | "°C"（度セルシウス）、"°F"（度ファーレンハイト）を区別。K（ケルビン）は°不要 | "25°C" → 「25°C」 |
| 許容差表現 | 「±」、「+X/-Y」等の記号を原文通り維持 | "±0.1 mm" → 「±0.1 mm」 |

### 図表・参照ラベルの形式
| Condition | Decision | Example |
|-----------|----------|---------|
| 図番号 (Figure X) | 「図X」に変換。文書全体で採番体系を統一 | "Figure 1" → 「図1」 |
| 表番号 (Table X) | 「表X」に変換。図とは別系統で採番 | "Table 2" → 「表2」 |
| 式番号 (Equation X) | 「式(X)」または「式X」に変換 | "Equation (1)" → 「式(1)」 |
| ステップ番号 (Step X) | 「手順X」または「STEP X」原文スタイルに応じて選択 | "Step 1" → 「手順1」 |
| 参照表現 ("as shown in Figure X") | 「図Xに示すように」日本語技術文書の定型表現に統一 | "as shown in Figure 3" → 「図3に示すように」 |
| クロスリファレンス ("see Section X") | 「第X節参照」または「X.X節を参照」文書内で統一 | "see Section 4.2" → 「4.2節を参照」 |
| 付録 (Appendix X) | 「付録X」または「附属書X」（JISでは「附属書」が正式） | "Appendix A" → 「附属書A」 |

### トラブルシューティング表の翻訳
| Condition | Decision | Example |
|-----------|----------|---------|
| 症状 (Symptom) | 「症状」または「現象」ユーザーマニュアルでは「症状」が一般的 | "Machine does not start" → 「機械が起動しない」 |
| 原因 (Cause) | 「原因」機械・電気・ソフトウェアの原因を区別しない一般用語 | "Power cord is disconnected" → 「電源コードが外れている」 |
| 処置 (Remedy / Action) | 「処置」または「対策」ユーザー向けでは「対処方法」も可 | "Connect the power cord securely" → 「電源コードを確実に接続する」 |
| 条件分岐 (If...then...) | 「…の場合：…する」箇条書き形式で条件と処置を対応 | "If error code E5 appears → Replace the filter" → 「エラーコードE5が表示された場合：フィルタを交換する」 |
| 凡例 (Legend / Key) | 「凡例」または「記号」設置場所の略号等は完全保持 | "S1: Main switch" → 「S1：主電源スイッチ」 |

### リスト・箇条書きの書式調和
| Condition | Decision | Example |
|-----------|----------|---------|
| 原文が箇条書き | 日本語訳でも原文の箇条書き形式を保持。マークダウン記号は原文に合わせる | 箇条書きの階層（- → * 等）を原文通り維持 |
| 導入句と各項目の文法構造 | 体言止め / 動詞で終止 / 名詞句を各項目で統一。混在を避ける | 全項目「…する。」で統一、または全項目「…の…」で統一 |
| 順序付きリスト | 番号付け方式（1. / (1) / ①）を原文に合わせるが、日本語では「1.」「(1)」「(a)」が一般的 | 日本語訳で番号体系を変更しない |
| リスト内の長い項目 | 必要に応じてサブパラグラフ化。ただし原文の情報階層を保持 | メイン項目 → インデント → サブ項目 の階層を維持 |

### 専門用語の初出明示
| Condition | Decision | Example |
|-----------|----------|---------|
| 専門用語が文書内で初出 | 原語（英語等）を括弧書きで併記。初出以降は和訳のみで可 | 「熱処理（heat treatment）」 |
| 造語や新しい概念 | 原語に加えて簡潔な定義または説明を併記 | 「フィールドバス（field bus：製造現場の機器間通信規格）」 |
| 業界で英語原語が標準 | カタカナ表記を基本とし、初出時のみ英語を併記 | 「イーサネット（Ethernet）」 |
| 英略語の初出 | 正式名称を初出時に併記。以降は略語のみで可 | 「API（Application Programming Interface）」→2回目以降は「API」 |

### 特許文書特有の処理
| Condition | Decision | Example |
|-----------|----------|---------|
| "comprising"（特許クレーム） | 「…からなる」「…を備える」クレームの構成要素列挙の定型訳 | "A device comprising: ..." → 「…からなる装置：」 |
| "characterized in that" | 「…を特徴とする」特許クレームの特徴付け表現 | "characterized in that the ratio is at least 2:1" → 「比率が少なくとも2：1であることを特徴とする」 |
| "means for"（ミーンズクレーム） | 「…手段」機能表現の定型訳 | "means for fastening" → 「固定手段」 |
| "embodiment" | 「実施形態」明細書での実装例の説明 | "In one embodiment" → 「一実施形態において」 |
| "prior art" | 「従来技術」特許審査での既存技術 | "This prior art reference discloses..." → 「この従来技術文献は…開示している」 |
| 特許番号・出願番号 | 番号体系を完全保持。国コード（JP, US, EP等）は保持 | "JP 2020-123456 A" → 「JP 2020-123456 A」 |

## Standard Conventions

### JIS規格体系と翻訳関連規格

**JIS規格の基本構造：**
- JIS XX XXXX : YYYY （部門記号 + 4桁番号 + 発行年）
  - 部門記号：A（土木建築）、B（機械）、C（電子機器）、D（自動車）、E（鉄道）、F（造船）、G（鉄鋼）、H（非鉄金属）、K（化学）、L（繊維）、M（鉱山）、P（パルプ・紙）、Q（管理システム）、R（窯業）、S（日用品）、T（医療）、W（航空）、X（情報処理）、Z（一般・基礎）
  - 例：JIS Z 8301:2019（規格票の様式及び作成方法）
  - 例：JIS B 0401-1:1998（寸法公差及びはめあいの方式）

**翻訳に直接関連するJIS規格：**
| 規格番号 | 名称 | 翻訳上の関連性 |
|----------|------|-----------------|
| JIS Z 8301 | 規格票の様式及び作成方法 | 技術文書の構成、用語、句読点、表記の基本規則を定義。翻訳書式の最重要参照規格 |
| JIS Z 8302 | 規格の構成及び作成に関する原則 | ISO/IEC Directives Part 2の対応JIS。規格文書構成の原則 |
| JIS Z 8202 | 量及び単位―記号（ISO 80000シリーズ対応） | SI単位記号の正しい表記法 |
| JIS Z 8203 | SI単位及びその組合せ | 単位の組み立て・表記法 |
| JIS Z 8303 | 図面の寸法記入方法 | 機械図面の寸法表記 |
| JIS Z 7253 | 安全データシート(SDS)作成 | 化学物質の安全管理文書の様式 |
| JIS Q 9001 | 品質マネジメントシステム | 品質管理関連の用語統一 |
| JIS Z 9101 | 安全標識のデザイン | 安全標示の標準ピクトグラム |
| JIS X 4051 | 日本語文書の組版方法 | 和文レイアウト、禁則処理、行長 |
| JIS B 0401 | 寸法公差及びはめあい | 公差表示の表記（+0.1/-0.2等） |
| JIS K 0211 | 分析化学用語（基礎部門） | 化学分析関連の用語統一 |
| JIS C 60068 | 環境試験方法（IEC 60068対応） | 環境試験関連の用語 |
| JIS C 0920 | 電気機器の筐体による保護等級（IPコード） | IPコード表記（IP65等）の保持 |

**ISO/IEC規格の日本語引用形式：**
- 「ISO 9001:2015（品質マネジメントシステム）」のように、番号 + 発行年 + 名称を併記
- 日本ではJISに採用されたISO規格は「JIS Q 9001：2015（ISO 9001:2015）」のように両規格番号を記載

**参照規格の表記統一：**
| 原文 | 日本語訳 | 備考 |
|------|----------|------|
| "per ISO 2768" | 「ISO 2768に従い」 | 日本語では「に従い」「により」「に準拠し」 |
| "as defined in JIS B 0401" | 「JIS B 0401で定義されているとおり」 | 受動態から能動態への変換例 |
| "meets the requirements of IEC 61000-4-2" | 「IEC 61000-4-2の要求事項を満たす」 | 規格要求への適合表現 |

### 日本語技術文書の書式標準

**句読点：**
- 横書き技術文書：JIS Z 8301は「，．」（コンマ＋ピリオド）を推奨
- 縦書き：伝統的に「、。」（読点＋句点）
- 近年のソフトウェア・IT文書では「、。」（読点＋句点）も広く使用される。文書全体で統一すること
- 箇条書きの末尾句点：各項目の末尾は「。」で統一、または「；」でつなぎ最終行のみ「。」とする

**数字表記：**
- 原則として算用数字（アラビア数字）を使用。ただし、慣用的な語（「一つ」「一般」等）は漢数字
- 桁区切り：3桁ごとにカンマ（,）または細スペース。小数点以下も3桁区切りしない
- 4桁以上の数字：カンマ区切りを原則。ただし年号は4桁そのまま（「2026年」）
- 漢数字の使用場面：契約書・法令・固有名詞（「第一節」等）

**英字表記：**
- 和文中の英単語は、前後に半角スペースを入れない（和文約物として処理）
- 英字と和文の間は、組版上の考慮が必要な場合は微調整するが、基本的にスペース不要
- 英字の大文字・小文字は原文に従う。ただし、文中で全て大文字の英文は適切に小文字化する

**見出し体系：**
- 章（Chapter） → 「第1章」または「1.」
- 節（Section） → 「1.1」
- 項（Subsection） → 「1.1.1」
- 目（Clause） → 「1.1.1.1」
- JIS Z 8301では「1.」「1.1」「1.1.1」の体系を推奨
- 見出しの末尾に句点は打たない（ぶら下げ見出し）

**参照記号：**
| 日本語 | 意味 |
|--------|------|
| 「→XX節」 | 参照先の指示 |
| 「（XX参照）」 | 参考情報 |
| 「前出」 | 同一文書内の既出箇所 |
| 「同上」「同左」 | 直前の項目と同じ |
| 「図Xに示す」 | 図の指示 |
| 「表Xのとおり」 | 表の指示 |

**数式の表記：**
- 文中の数式：変数はイタリック体が基本だが、プレーンテキストでは区別しない
- 独立行の数式：センター揃え、式番号は右端に「(1)」形式
- 変数の説明：「ここで、xは…、yは…」の形式
- ギリシャ文字：α、β、γ、μ、Ω等、日本語で一般的な読みを併記してもよいが、記号は原文通り
- ベクトル表記：太字または矢印記号を保持。プレーンテキストでは区別が困難な場合注記
