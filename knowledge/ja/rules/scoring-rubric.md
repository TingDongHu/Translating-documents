# 日本語翻訳品質評価ルーブリック (Japanese Translation QA Scoring Rubric)

評価尺度：各次元 0～5 点。加重合計から最終スコア（百分率）を算出します。各次元の重みは次元ごとに定義されています。

---

## 全体制約 (Global Constraints)

- **重大な誤り（Critical Error）が1つでもある場合** → 最終スコアは最高 60 点（D）。
- 重大な誤りの定義：
  - 数値誤り（金額・日付・数量の誤変換）
  - 意味の逆転（肯定↔否定の誤り）
  - 敬語レベルの根本的な取り違え（尊敬語が必要な箇所にタメ口）
  - 元号の誤記（例：令和6年を平成6年と表記）
  - 原文にない内容の追加または原文にある内容の脱落（重要なもの）
- **採点不能（N/A）**：次元が適用できない場合（例：OCRが関係しない文書で次元6はN/A）は評価対象外とし、重みを再配分する。

---

## Dimension 1: 数値的正確性 (Numerical Accuracy) — 重み 30%

**チェック項目：**
- 小数点と桁区切りが日本語規則に従っているか（小数点は `.`、桁区切りは `,`）
- マイナス表記が正しいか（`−` または `-`）
- パーセント記号の位置が正しいか（`5%`、スペースなし）
- 金額の表記が正しいか（`¥1,000` または `1,000円`）
- 和暦と西暦の換算が正しいか（例：令和6年 = 2024年、平成30年 = 2018年）
- 日付形式が日本語規則に従っているか（`2026年5月24日` または `2026/05/24`）
- 時刻が24時間表記になっているか

| 点数 | 基準 | 解釈 |
|------|------|------|
| 5 | すべての数値・通貨・日付・時刻が日本語規則に100%準拠。換算も含めて完全に正確。 | 完璧 |
| 4 | 軽微な問題が1～2件（例：桁区切りコンマの1箇所の欠落、日付のゼロ埋め揺れ）。意味に影響なし。 | 良好 |
| 3 | 複数の形式誤り（3～5箇所）：桁区切りが複数箇所で誤っている、パーセント位置が不統一など。 | 普通 |
| 2 | 体系的な誤り：小数点にコンマを使用（`3,14`）、日付が米国式（`05/24/2026`）、または元号を誤変換（複数箇所）。 | 不良 |
| 1 | 大半の数値形式が未変換（原文の英語形式のまま）、和暦変換がほぼ行われていない。 | 不十分 |
| 0 | 数値形式の変換が全くなされていない。主要な日付・金額が誤っている。 | 不可 |

**ゼロトレランス（この次元で即座に 0 点）：**
- 元号（和暦）の誤変換 → 令和6年を「平成6年」とするなど
- 金額の誤記 → ¥1,000,000 を ¥100,000 と誤記
- 日付の誤変換 → 2024年5月24日を 2024年5月25日 と誤記

---

## Dimension 2: 用語の一貫性 (Terminology Consistency) — 重み 20%

**チェック項目：**
- 同一用語が文書全体で統一されているか（カタカナ揺れの確認）
- 固有名詞の表記が適切か（人名・地名・企業名のカタカナ転写）
- 専門用語の訳が一貫しているか
- 「ですます調」と「である調」が混在していないか
- 日本語の定訳が存在する用語に適切な訳語を当てているか

| 点数 | 基準 | 解釈 |
|------|------|------|
| 5 | 全用語が文書全体で完全に一貫。固有名詞の表記も適切。文体の混在なし。 | 完璧 |
| 4 | 軽微な用語揺れが1～2箇所（例：「ユーザー」と「利用者」が混在、固有名詞の長音記号の有無の揺れ）。 | 良好 |
| 3 | 用語揺れが3～5箇所（例：同一概念に3種類以上の訳語、固有名詞の表記ゆれが複数）。 | 普通 |
| 2 | 体系的な用語不統一。文体（である調／ですます調）が章ごとに混在。専門用語の誤訳が散見。 | 不良 |
| 1 | 用語が大きく不統一。訳語選択に体系性がない。原文の専門用語がほぼそのまま残っている。 | 不十分 |
| 0 | 用語の一貫性が完全に欠如。文体も無秩序。意味が取れないレベル。 | 不可 |

**カタカナ揺れの許容範囲：**
- 許容される揺れ（軽微）：
  - `コンピュータ` vs `コンピューター`（長音省略は慣用的に許容される場合あり）
  - `サーバ` vs `サーバー`（同上）
  - `インターネット` vs `インターネット`（表記ゆれがあれば統一を推奨するが、致命的ではない）
- 許容されない揺れ（重大）：
  - `アクセス` vs `接近`（異なる訳語）
  - `削除` vs `デリート` vs `消去`（同一操作に対する不統一）
  - `ユーザー` vs `使用者` vs `利用者`（同一文脈での混在）
- 文体の混在（である調とですます調）は、明らかな意図（引用など）がない限り重大な問題とする。

---

## Dimension 3: 意味的正確性 (Semantic Fidelity) — 重み 25%

**チェック項目：**
- 原文の意味が正確に伝わっているか（中国語→日本語、英語→日本語の誤訳チェック）
- 肯定・否定・疑問のモダリティが正しいか
- 数量・割合の表現が原文と一致しているか
- 原文のニュアンス（皮肉・強調・留保）が適切に再現されているか
- 日本語として自然でありながら、原文の意図を正確に反映しているか

| 点数 | 基準 | 解釈 |
|------|------|------|
| 5 | 原文の意味が完全かつ正確に伝達されている。ニュアンスも適切に再現。日本語として自然。 | 完璧 |
| 4 | 軽微な意味のずれが1～2箇所（例：副詞の強弱がやや異なる、やや不自然だが意味は通じる）。 | 良好 |
| 3 | 複数の意味のずれ（3～5箇所）。ただし文書全体の理解に支障はないレベル。 | 普通 |
| 2 | 意味の誤解が複数（肯定↔否定の誤りを含む場合あり）。原文の意図から大きく乖離した箇所がある。 | 不良 |
| 1 | 意味の重大な誤りが頻出。原文の内容が大きく損なわれている。 | 不十分 |
| 0 | 意味が完全に異なっている。原文の意図がまったく伝わらない。 | 不可 |

**ゼロトレランス（この次元で最低 1～2 点に引き下げ）：**
- 肯定と否定の誤変換（例：`~できない` を `~できる` と訳す）
- 数量（多い／少ない）の逆転
- キーフレーズの欠落により原文の主張が変わってしまう場合

**中国語→日本語特有の注意点：**
- 中国語の `手紙`（トイレットペーパー）を日本語の `手紙`（手紙）と誤訳しないこと
- 中国語の `勉強`（無理強い）を日本語の `勉強`（学習）と誤訳しないこと
- 中国語の `大丈夫`（立派な男）を日本語の `大丈夫`（問題ない）と誤訳しないこと
- 中国語の `走`（歩く）を日本語の `走`（走る）と誤訳しないこと
- 中国語の `的` をそのまま `的` としない（日本語の `の` に変換）

**英語→日本語特有の注意点：**
- `would like to` → 「したいと思う」ではなく「〜したい／いたしたい」
- `should be able to` → 「できるべき」ではなく「できるはず」
- 英語の現在完了 → 日本語の「〜た」または「〜ている」（文脈判断）
- 英語の受動態 → 日本語の能動態に変換することを検討

---

## Dimension 4: 書式準拠 (Format Compliance) — 重み 10%

**チェック項目：**
- 日本語の句読点（句点 `。`、読点 `、`）が正しく使用されているか
- 括弧の種類が日本語規則に従っているか（`「」` `『』` `（）`）
- 中黒（`・`）の使い方が適切か
- 波線（`〜`）が範囲を示すために正しく使われているか
- 三点リーダー（`…`）やダッシュ（`——`）が正しく使用されているか
- 見出し・箇条書きの書式が日本語規則に従っているか
- 単位と数字の間のスペースが適切か（原則としてスペースなし）
- 表の書式が日本語の慣習に従っているか

| 点数 | 基準 | 解釈 |
|------|------|------|
| 5 | 日本語の書式規則に100%準拠。句読点・括弧・記号がすべて正確。 | 完璧 |
| 4 | 軽微な書式上の問題が1～2箇所（例：中黒と読点の使い分けに一貫性がない、三点リーダーの数が不統一）。 | 良好 |
| 3 | 書式の問題が3～5箇所（例：かぎ括弧の代わりにクォーテーションマークを使用、数字と単位の間に余分なスペース）。 | 普通 |
| 2 | 体系的な書式問題（例：英文の句読点 `,` `.` を日本語文中に使用、括弧の種類が日本語でない）。 | 不良 |
| 1 | 日本語の書式規則がほとんど適用されていない（英文のピリオド・コンマをそのまま使用）。 | 不十分 |
| 0 | 日本語の書式規則が全く適用されていない。原本の書式をそのまま維持。 | 不可 |

**ゼロトレランス：**
- 句点（。）の欠如による文章の区切り不明瞭が複数箇所
- かぎ括弧（`「」`）を使用すべき箇所に英文クォーテーションマーク（`"`）をそのまま使用（技術文書タイトル等、意図的な場合を除く）

---

## Dimension 5: 完全性 (Completeness) — 重み 10%

**チェック項目：**
- 原文のすべての文・段落・節が翻訳されているか
- 原文の表・図表・キャプションが翻訳または適切に処理されているか
- 脚注・ヘッダー・フッターの翻訳漏れがないか
- 原文の構成（箇条書きの項目数、段落数）が維持されているか
- 翻訳による内容の追加（原文にない情報の挿入）がないか
- HTMLタグ・マークダウンタグ・プレースホルダーなどの構造要素が保持されているか

| 点数 | 基準 | 解釈 |
|------|------|------|
| 5 | 原文の内容が完全に網羅されている。欠落・追加なし。構造要素も完全に保持。 | 完璧 |
| 4 | 1～2の軽微な欠落（例：キャプション1行の未翻訳、脚注の見落とし）。内容への影響は軽微。 | 良好 |
| 3 | 複数の欠落（3～5箇所）。1段落程度の欠落があるが、全体の理解への影響は限定的。 | 普通 |
| 2 | 重要な欠落が複数（例：箇条書きの1項目全体が欠落、表の1行が未翻訳）。 | 不良 |
| 1 | 大量の欠落。文書の半分程度しか翻訳されていない。 | 不十分 |
| 0 | ほぼ未翻訳。原文のまま。 | 不可 |

**ゼロトレランス：**
- 原文の段落または節全体の欠落（1段落以上）
- 重要な数値データを含む表の行の欠落
- 法的効力を持つ条項（契約書の条項など）の欠落

---

## Dimension 6: 推論／OCR品質 (Inference / OCR Quality) — 重み 5%

**チェック項目：**
- OCR由来の文字認識誤りが修正されているか
- 不確かなOCR結果が `[復元: ...]` 形式で適切にマークされているか
- 日本語OCR特有の誤認識（長音と漢字の「一」の混同、濁点半濁点の誤りなど）が修正されているか
- 旧字体が現代文では新字体に統一されているか（歴史的文書を除く）
- 禁則処理（行頭禁則・行末禁則）が適切に行われているか
- ソフトハイフンなど不要な制御文字が除去されているか

| 点数 | 基準 | 解釈 |
|------|------|------|
| 5 | OCR誤りが完全に修正されている。不確かな文字は適切に `[復元: ...]` マーク。旧字体・禁則処理も正確。 | 完璧 |
| 4 | 軽微なOCR関連問題が1～2箇所（例：明らかな誤字1箇所の見落とし、`[復元]` マークの付け忘れ1箇所）。 | 良好 |
| 3 | 3～5箇所のOCR関連問題（例：長音と漢字の「一」の混同が数箇所、旧字体が複数未修正）。 | 普通 |
| 2 | 体系的なOCR問題が未解決（例：濁点の誤りが頻発、`[復元]` マークがほとんどない）。 | 不良 |
| 1 | OCR問題が広範囲に残存。原文の判読に支障をきたすレベル。 | 不十分 |
| 0 | OCR処理が全く行われていない（生のOCR出力のまま）。 | 不可 |

**ゼロトレランス：**
- 判読不能な箇所を `[復元]` なしで放置（特に固有名詞・数値）
- OCR誤認識により日付・金額が変わってしまっている箇所が未修正
- `の` と `O` の混同など、明らかな誤認識が放置されている

**OCR品質報告：**
- 文書全体のOCR品質が低い場合、冒頭に注記を入れること：
  ```
  [OCR品質: 低下 — 原本に多数の認識不良あり。特に数字・長音記号に注意。]
  ```

## 加重スコア計算

| 次元 | 重み | 配点 (0-5) | 加重点 =
重み × 配点 |
|------|------|-----------|---------------------------|
| 1. 数値的正確性 | 30% | 0~5 | 30% × 配点 |
| 2. 用語の一貫性 | 20% | 0~5 | 20% × 配点 |
| 3. 意味的正確性 | 25% | 0~5 | 25% × 配点 |
| 4. 書式準拠 | 10% | 0~5 | 10% × 配点 |
| 5. 完全性 | 10% | 0~5 | 10% × 配点 |
| 6. OCR品質 | 5% | 0~5 | 5% × 配点 |

**最終スコアの計算式：**

```
加重合計 = (D1×0.30) + (D2×0.20) + (D3×0.25) + (D4×0.10) + (D5×0.10) + (D6×0.05)

最終スコア(%) = (加重合計 / 5) × 100
```

**例：** D1=4, D2=5, D3=4, D4=3, D5=5, D6=4
```
加重合計 = (4×0.30) + (5×0.20) + (4×0.25) + (3×0.10) + (5×0.10) + (4×0.05)
        = 1.20 + 1.00 + 1.00 + 0.30 + 0.50 + 0.20
        = 4.20
最終スコア = (4.20 / 5) × 100 = 84%
```

## 最終評価グレード

| スコア範囲 | グレード | 評価 | 対応 |
|-----------|---------|------|------|
| 90～100% | A | 優秀 | 軽微な修正のみで承認可。用語の統一確認のみ。 |
| 80～89% | B | 良好 | 軽微な修正を推奨。要修正項目リストを作成。 |
| 70～79% | C | 普通 | 修正が必要。中程度の改訂を推奨。 |
| 60～69% | D | 不良 | 大幅な修正が必要。主要な問題を洗い出し再翻訳を検討。 |
| 0～59% | F | 不可 | 全面的な再翻訳が必要。 |

### 合格基準 (Acceptance Criteria)
- **全次元で最低 2 点以上**であること（どの次元も 0 点または 1 点がないこと）。
- **次元1（数値的正確性）は最低 3 点以上**であること（数値誤りはゼロトレランス）。
- 最終スコア **70% 以上**を合格とする。
- 法的文書・金融文書・医療文書の場合：
  - 次元1（数値的正確性）は **4 点以上**必要。
  - 次元3（意味的正確性）は **4 点以上**必要。
  - 総合スコア **80% 以上**を合格とする。

### 重大エラーによるスコア上限
| 重大エラーの種類 | 該当する場合の上限 |
|-----------------|-------------------|
| 数値誤り（金額・日付） | 60%（D） |
| 意味の逆転（肯定↔否定） | 60%（D） |
| 和暦元号の誤記 | 60%（D） |
| 敬語レベルの根本的誤り | 60%（D） |
| 重要な内容の欠落（節単位） | 60%（D） |
| 複数の重大エラーが同時発生 | 50%（F） |
