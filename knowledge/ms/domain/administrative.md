## Reader Model

**Who reads this document and what do they expect?**

Administrative document readers in Malaysia are government officers, civil servants, policy makers, local authority staff, and citizens interacting with government services. They read the document to understand regulations, follow procedures, or fulfill bureaucratic requirements. They expect:

- **Absolute clarity** — administrative documents must be unambiguous. Citizens and officers must understand exactly what is required.
- **Standardized format** — government correspondence follows established Malaysian administrative conventions that are deeply familiar to civil servants.
- **Bahasa Baku** — government documents must use standard formal Malay as prescribed by Dewan Bahasa dan Pustaka (DBP).
- **Proper authority references** — documents must correctly reference the relevant Acts, regulations, and government circulars.
- **Cultural appropriateness** — Malaysia's multicultural context requires inclusive language that respects all communities.

**What would break their trust?**
- Incorrect authority references (wrong Act number, wrong ministry name).
- Informal or colloquial language in an official document.
- Inconsistent formatting that violates government document conventions.
- Missing or incorrect official letterhead elements.

## Decision Framework

### 1. Government Correspondence Structure

| Element | Malay Convention | Notes |
|---|---|---|
| Letterhead | Logo + Nama Kementerian/Jabatan + Alamat | Official header |
| Reference number | `Ruj: [Code]/[Year]` | Unique tracking number |
| Date | `Tarikh: 27 Mei 2026` or `DD/MM/YYYY` | Formal date format |
| Addressee | `Tuan/Puan [Name]`, `[Position]`, `[Department]` | Formal address |
| Salutation | `Dengan segala hormatnya` or `Saya yang menurut perintah` | Opening formula |
| Subject | `Perkara: [Subject]` | Underlined or bold |
| Body | Formal, passive-heavy, directive tone | Administrative style |
| Closing | `Sekian, terima kasih` or `Dengan ini saya menyerahkan` | Formal close |
| Signature | `Yang menurut perintah`, Name, Position | Authority line |
| CC | `s.k.` (salinan kepada) | Distribution list |
| Appendix | `Lampiran:` | Supporting documents |

## Error Pattern Library

### Critical Errors (zero tolerance)

| Error | Why It Is Critical |
|---|---|
| **Wrong ministry/department name** | Using an outdated or incorrect government body name undermines the document's authority. Always verify current names. |
| **Incorrect Act/Regulation number** | A wrong section reference can invalidate the legal basis of the document. Verify against current legislation. |
| **Informal language in official correspondence** | Using `nak` instead of `ingin`, or `dah` instead of `telah`, in a government letter is a serious register violation. |
| **Wrong title for official** | Using `Datuk` where `Dato'` is correct, or using the wrong state honorific, is a significant protocol error. |

### Common Errors (frequent in practice)

| Error | Guidance |
|---|---|
| **Incorrect reference number format** | Government reference numbers follow specific codes. Verify the format with the issuing department. |
| **Passive voice misuse** | Government Malay heavily uses `di-` passive: `Surat ini dirujuk` (not `Surat ini merujuk`). Ensure the passive construction is correct. |
| **Missing `Perkara:` subject line** | Administrative letters require a subject line. Always include `Perkara:` followed by the subject. |
| **Inconsistent date format** | Government documents should use DD/MM/YYYY consistently. Do not mix formats. |

### Subtle Errors (easy to miss)

| Error | Guidance |
|---|---|
| **Wrong `s.k.` usage** | `s.k.` (salinan kepada) is for distribution, not for the main addressee. Ensure it is used correctly. |
| **Missing authority phrase** | `Saya yang menurut perintah` or `Dengan segala hormatnya` should open official correspondence. Omission is unusual. |
| **Incorrect `Ruj:` format** | Reference numbers must match the department's coding system. |
