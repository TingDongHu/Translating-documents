## Reader Model (독자 모델)

**누가 이 문서를 읽고 무엇을 기대하는가?**

상업 문서의 독자는 경영진, 계약 담당자, 영업 사원, 구매 담당자, 무역 실무자로서 문서를 통해 비즈니스 의사결정을 내린다. 다음을 기대한다:

- **명확한 권리와 의무** — 각 당사자의 책임, 지급 조건, 납품 일정이 명확히 명시되어야 한다. 모호함은 분쟁의 원인이 된다.
- **전문 용어의 정확성** — 무역 규칙(Incoterms), 지급 조건(L/C, T/T), 보증 조건 등 업계 표준 용어를 정확히 사용해야 한다.
- **적절한 격식 수준** — 한국 비즈니스 문화에서는 상대방과의 관계에 따라 존댓말(합쇼체/하오체)과 반말(해라체)의 구분이 중요하다. 격식 수준의 오류는 무례함으로 비춰질 수 있다.
- **일관된 회사명/제품명** — 회사명, 브랜드명, 제품명은 모든 문서에서 동일하게 표기되어야 한다.
- **숫자 정보의 정확성** — 금액, 수량, 일정 등 숫자 정보는 단 1자의 오류도 허용되지 않는다.

**무엇이 신뢰를 무너뜨리는가?**
- Incoterms 조건 오번역으로 인한 비용/위험 부담 오해
- 지급 조건 오기로 인한 대금 회수 문제
- 회사/제품명 불일치로 인한 혼선
- 한국 비즈니스 예의에 어긋나는 부적절한 경어법 사용
- 날짜 및 기한 오번역으로 인한 계약 위반

## Decision Framework (의사결정 체계)

### D1: Incoterms 및 무역 조건

| 조건 | 결정 |
|------|------|
| Incoterms (FOB, CIF, EXW 등) | 영문 약어 그대로 유지. 설명이 필요하면 괄호로 추가. 한국 무역 실무에서 Incoterms는 영문으로 통용 |
| 지급 조건 (L/C, T/T, D/P, D/A) | 영문 약어 유지. 첫 사용 시 약어 풀어쓰기. `L/C (Letter of Credit, 신용장)` |
| 선적 조건 | `partial shipment` → `분할 선적`, `transshipment` → `환적` |
| 포장 조건 | `export packing` → `수출 포장`, `sea-worthy packing` → `해상 운송용 포장` |
| 검수 조건 | `inspection` → `검사` 또는 `검수`, `pre-shipment inspection` → `선적 전 검사` |

### D2: 지급 및 금액 조건

| 조건 | 결정 |
|------|------|
| 금액 표기 | 숫자와 한글 병기. `USD 50,000 (금 오만 미달러정)` |
| 선급금/잔금 | `advance payment` → `선급금`, `down payment` → `계약금`, `balance` → `잔금` |
| 할인 조건 | `early payment discount` → `조기 지급 할인`, `trade discount` → `거래 할인` |
| 연체 이자 | `late payment interest` → `연체 이자` 또는 `지연 이자`, `default interest` → `지체 이자` |
| 부가가치세 | `VAT` → `부가가치세(VAT)` — 첫 사용 시 약어 설명 후 `VAT` 또는 `부가세`로 통일 |
| 통화 약어 | ISO 4217 통화 코드 사용: KRW, USD, EUR, JPY 등 |
| 원화 표기 | `₩50,000` 또는 `금 오만 원정` (법률 효력이 있는 문서는 한글 금액 병기) |

### D3: 계약 조항 번역

| 조건 | 결정 |
|------|------|
| 계약 기간 | `effective date` → `발효일`, `term` → `계약 기간`, `renewal` → `갱신`, `termination` → `종료` 또는 `해지` |
| 해지 조건 | `termination for cause` → `귀책 사유에 의한 해지`, `termination for convenience` → `편의에 의한 해지` |
| 배상 책임 | `limitation of liability` → `책임의 제한`, `indemnification` → `손해배상` |
| 분쟁 해결 | `governing law` → `준거법`, `arbitration` → `중재`, `jurisdiction` → `관할` |
| 비밀 유지 | `confidential information` → `기밀 정보`, `non-disclosure` → `비밀 유지` |
| 양도 | `assignment` → `양도`, `novation` → `경개(更改)` (계약 인수) |
| 불가항력 | `force majeure` → `불가항력`, 한국 계약서에서는 영문 `Force Majeure`도 관행적으로 사용 |

### D4: 비즈니스 서신 및 이메일

| 조건 | 결정 |
|------|------|
| 공식 서신 (제안서, 견적서) | `합쇼체` (합니다 체) 사용 — 가장 격식 높은 문체 |
| 업무 이메일 | `하오체` (합니다/요 체) — 일반적 격식 유지 |
| 내부 메모 | `해라체` (한다 체) — 수직적 관계에 따라 변동 가능 |
| 호칭 | `담당자 귀하`, `사장님`, `팀장님` — 한국 비즈니스 호칭 체계 준수 |
| 맺음말 | `감사합니다.`, `수고하십시오.` — 표준 비즈니스 맺음말 사용 |
| 직함 번역 | `CEO` → `대표이사`, `Manager` → `팀장` 또는 `과장` (회사 체계에 따라 조정) |

### D5: 마케팅 및 홍보 자료

| 조건 | 결정 |
|------|------|
| 슬로건/캐치프레이즈 | 창의적 번역 허용. 원문의 의도와 감흥을 살리는 현지화 우선. 직역 금지 |
| 제품 특징 설명 | 사실 정보는 정확히 유지. 감성적 표현은 한국 시장에 맞게 조정 |
| 소비자 대상 문구 | `합니다체` 또는 `요체` 사용 — 친근하면서도 신뢰감 있는 어조 |
| 브랜드 슬로건 | 기존 한국어 공식 슬로건이 있는 경우 반드시 사용. 없는 경우 브랜드 톤앤매너에 맞게 번역 |
| 가격 표시 | 한국식 가격 표기법(세금 별도/포함 명시) 준수. `부가세 별도` / `VAT 포함` |
| 법적 고지/면책 문구 | 정확한 번역 필수. 마케팅 문구와 별도로 정확한 법적 표현 사용 |

### D6: 회사 및 제품명

| 조건 | 결정 |
|------|------|
| 회사명 | 원어 그대로 유지. 한국 등기 회사명이 있는 경우 해당 명칭 사용 |
| 브랜드명 | 원어 유지. 한국어 발음이 공식화된 경우 병기 가능. 예: `Samsung (삼성)` |
| 제품 모델명/코드 | 대소문자, 숫자, 특수문자 포함 정확히 원형 유지 |
| 상표 기호 | `®`, `™` 원형 유지 |
| 법인 구분 표시 | `(주)`, `(유)`, `Inc.`, `Ltd.`, `GmbH` — 원문 법인 표기 유지가 원칙 |
| 한국 법인 표기 | `㈜삼성전자` 형식. 외국 회사는 `ABC Inc.` 유지 |

### D7: 상업 문서 날짜 및 기한

| 조건 | 결정 |
|------|------|
| 납품 기한 | `delivery date` → `납품일`, `lead time` → `리드 타임` 또는 `납기` |
| 지급 기한 | `payment due date` → `지급 기일` 또는 `결제 기일` |
| 서류 제출 기한 | `deadline` → `마감일`, `closing date` → `마감일` |
| 유효 기간 | `validity period` → `유효 기간`, `offer valid until` → `...까지 유효` |
| 영업일 기준 | `business day` → `영업일`, `calendar day` → `역일(曆日)` 또는 `달력일` |
| 날짜 표기 | `YYYY년 MM월 DD일` 형식. 영문 병기 시 `April 30, 2025` |

### D8: 입찰 및 제안

| 조건 | 결정 |
|------|------|
| 입찰 공고 | `RFP (Request for Proposal)` → `제안 요청서` 또는 `RFP 유지`, `RFQ` → `견적 요청서` |
| 제안서 구조 | `executive summary` → `경영진 요약`, `scope of work` → `작업 범위`, `methodology` → `수행 방법론` |
| 평가 기준 | `evaluation criteria` → `평가 기준`, `weighted score` → `가중치 점수` |
| 자격 요건 | `qualification requirements` → `자격 요건`, `eligibility` → `응찰 자격` |
| 보증 | `bid bond` → `입찰 보증금`, `performance bond` → `이행 보증금`, `advance payment bond` → `선급금 보증서` |

## Standards (표준 및 참조)

### 주요 무역 및 상업 법규

| 법규/규칙 | 영문 명칭 | 적용 분야 |
|-----------|-----------|----------|
| **상법** | Commercial Act | 상행위, 회사, 보험, 해상 |
| **대외무역법** | Foreign Trade Act | 수출입 무역 |
| **관세법** | Customs Act | 통관, 관세 |
| **상업등기법** | Commercial Registration Act | 법인 등기 |
| **약관규제법** | Act on the Regulation of Terms and Conditions | 표준 약관 |
| **국제물품매매계약에관한UN협약(CISG)** | CISG | 국제 물품 매매 |
| **신용장통일규칙(UCP 600)** | Uniform Customs and Practice for Documentary Credits | 신용장 거래 |
| **Incoterms** | International Commercial Terms | 무역 조건 해석 |

### 한국 상업 문서 작성 관행

| 문서 유형 | 한국어 명칭 | 특징 |
|-----------|-------------|------|
| 견적서 | Quotation / 견적서 | 숫자 정확성, 유효 기간 명시 |
| 주문서 | Purchase Order / 주문서 | 주문 번호, 품목 코드 |
| 납품서 | Delivery Note / 납품서(거래명세서) | 납품일자, 품목별 수량 |
| 세금계산서 | Tax Invoice / 세금계산서 | 부가가치세법 규정 준수 필수 |
| 계약서 | Contract / 계약서 | 조문 체계, 당사자 정보, 금액, 날짜 |
| 발주서 | Order Sheet / 발주서 | 구매 부서 발행 공식 문서 |
| 인수증 | Receipt / 인수증 | 물품 인수 확인 |

### 비즈니스 커뮤니케이션 경어법 체계

| 상황 | 문체 | 예시 |
|------|------|------|
| 최고 경영자/대표이사에게 | 합쇼체 (가장 높은 격식) | `검토하여 주시기 바랍니다.` |
| 부서장/팀장에게 | 합쇼체 | `보고 드립니다.` |
| 동료/협력사 담당자 | 하오체 (`요`체) | `확인 부탁드립니다.` |
| 하청업체/후배 | 하게체 또는 해라체 | `진행 바랍니다.` |
| 공식 문서 | 합쇼체 | `이에 제안합니다.` |

### 주요 주의사항

1. **영문 상업 용어의 한글 표기** — 한국 무역 실무에서 널리 통용되는 영문 용어(FOB, L/C, B/L, AWB 등)는 그대로 유지
2. **금액 표기의 이중 안전장치** — 숫자와 한글 금액을 모두 기재하여 오기(誤記) 방지
3. **계약서 문체 일관성** — `~한다` 체 유지. 한 문서 내에서 문체 혼용 금지
4. **한국 사업자 등록 번호** — 외국 회사의 경우 `business registration number` → `사업자 등록 번호` (필요시 해당 국가 등록 번호 설명 추가)
5. **도장(날인)에 대한 설명** — 한국의 도장(인감) 문화를 감안한 번역 필요. `seal` → `도장(날인)` / `signed and sealed` → `서명 및 날인`

## Error Pattern Library (오류 패턴 라이브러리)

### Critical Errors (제로 허용 오류)

| Error | Why It Is Critical |
|-------|-------------------|
| **Incoterms 조건 오번역** | Incoterms는 비용, 위험, 책임을 배분하는 표준 무역 조건. FOB를 CIF로 오역하면 운송비와 보험료 부담 주체가 변경되어 금전적 손실 발생. |
| **지급 조건 오류** | L/C를 T/T로 혼동하거나 지급 기일을 잘못 번역하면 대금 회수 불가 또는 계약 위반으로 이어짐. |
| **금액 표기 오류** | 단가, 총액, 할인율 등 금액 정보의 불일치는 계약서 신뢰성을 즉시 무너뜨림. 숫자와 한글 금액이 일치하는지 반드시 확인. |
| **회사명/법인 형태 오역** | 외국 회사명 뒤의 Ltd., Inc., GmbH 등을 한국어 법인 형태로 변경하는 것은 법인격 식별을 어렵게 함. 원래 형태 유지. |
| **납품 기한 오역** | "within 30 days"를 "30일 이내"가 아닌 "30일 이후"로 오역하면 계약 위반. 기한 표현은 가장 엄격히 검증. |
| **통화 오류** | USD, KRW, EUR, JPY 등 ISO 통화 코드 혼동은 금액의 실제 가치를 왜곡. 달러($)만으로 표기된 경우 어떤 통화인지 확인 필요. |

### Common Errors (빈번한 오류)

| Error | Guidance |
|-------|----------|
| **shall/must/may 혼용** | shall(~해야 한다)은 계약상 의무, must(~하여야 한다)는 법적 요구사항, may(~할 수 있다)는 재량 권한. 혼용 금지. |
| **비즈니스 서신 경어법 오류** | 한국어 비즈니스 서신에서는 상대방과의 관계에 따른 경어법 선택이 중요. 첫 연락 시 항상 최고 격식(합쇼체) 사용. 관계가 가까워져도 공식 문서는 격식 유지. |
| **제품 사양 오역** | 기술적 사양(용량, 크기, 재질, 성능)은 정확히 번역. 마케팅적 표현과 기술적 표현을 구분. |
| **견적서/주문서 형식 무시** | 견적서, 주문서, 세금계산서 등 각 상업 문서는 고유한 필수 항목이 있음. 항목 누락 시 법적 효력 문제 발생 가능. |
| **부가가치세(VAT) 처리 오류** | 한국은 부가세가 10%이며 가격 표시 시 "부가세 별도" 또는 "VAT 포함"을 명시해야 함. 한국 세법에 따른 VAT 처리 방식 반영. |

### Subtle Errors (놓치기 쉬운 오류)

| Error | Guidance |
|-------|----------|
| **숫자 형식 혼동** | 한국어는 천 단위 쉼표 사용(1,000,000). 일부 유럽 국가는 점을 천 단위 구분자로 사용. 원본의 천단위/소수점 구분자를 반드시 확인. |
| **한국 비즈니스 호칭 체계 오해** | 한국어 "대리"는 직급명(Assistant Manager)이지 대리인(agent)이 아님. 한국 회사 직급 체계(사원-대리-과장-차장-부장)에 맞게 번역. |
| **선적/운송 문서 용어 오류** | B/L(선하증권), AWB(항공화물운송장), S/O(선적지시서) 등 운송 문서는 각각 고유한 법적 기능이 있음. 정확히 구분 번역. |
| **할인 조건 표현** | "2/10, n/30" 조건은 "10일 내 지급 시 2% 할인, 30일 내 전액 결제"로 번역. 한국식 "2/10, n/30"과 동일한 관행이므로 설명문 번역에 주의. |
| **관세/통관 용어 오류** | "customs clearance" → "통관", "customs duty" → "관세", "tariff" → "관세율", "bonded warehouse" → "보세창고". 각 용어의 정확한 한국어 대응어 사용. |
| **수량 단위 오류** | 한국은 "EA"를 개수 단위로 사용. "PCS"도 사용되나 문서 내 일관성 유지 필요. "BOX", "SET", "KG" 등 포장 단위도 통일. |
