# Domain: Commercial (Bisnis / Niaga) — Indonesian

## 1. Reader Model: How to Read Indonesian Commercial Documents

### 1.1 Macro-Level Architecture

Indonesian commercial documents blend formal Indonesian business conventions with Dutch-influenced correspondence structures and modern international business practices. Documents range from highly formal (surat perjanjian niaga) to transactional (faktur, kuitansi, purchase order).

**Typical business correspondence structure (surat niaga):**

```
    [Kop Surat / Letterhead] — Nama perusahaan, alamat, telepon, NPWP
        → [Nomor / Reference Number] — Format: No. XXX/YYY/ZZZ
            → [Lampiran / Attachment(s)] — "Lamp.: X lembar"
                → [Hal / Subject] — "Hal: [subject line]" (bold or underlined)
                    → [Tanggal / Date] — "X [Bulan] YYYY"
                        → [Kepada Yth. / To] — Alamat penerima
                            → [Salam Pembuka / Salutation] — "Dengan hormat," / "Yth."
                                → [Isi Surat / Body]
                                    → [Salam Penutup / Closing] — "Hormat kami,"
                                        → [Tanda Tangan / Signature]
                                            → [Tembusan / Cc] — "Tembusan:"
```

**Typical commercial invoice structure (faktur penjualan):**

```
    [FAKTUR PENJUALAN / SALES INVOICE] — Header
        → [Nomor Faktur / Invoice Number]
            → [Tanggal Faktur / Invoice Date]
                → [Data Penjual / Seller Info] — Nama, alamat, NPWP
                    → [Data Pembeli / Buyer Info] — Nama, alamat, NPWP
                        → [Tabel Barang/Jasa / Items Table]
                            → [Jumlah Harga / Subtotal]
                                → [Pajak PPN / VAT] — 11% (per UU HPP 2022)
                                    → [Pajak PPh / Withholding Tax] — Jika ada
                                        → [Total / Grand Total]
                                            → [Terbilang / Amount in Words]
                                                → [Syarat Pembayaran / Payment Terms]
```

### 1.2 Meso-Level Patterns

**Move 1: Salutation and Opening**
- "Yth." (Yang terhormat) = "Dear" / "To the esteemed"
- "Dengan hormat," = "Dear Sir/Madam," or "With respect,"
- Never informal in first contact; tone must be maintained

**Move 2: Subject Line (Perihal)**
- Placed after recipient address, before salutation
- Usually bold and/or underlined
- Should be concise but descriptive

**Move 3: Body — Persuasive-Opening to Action-Closing**
- Indonesian commercial writing tends to be **more indirect** than English:
  - Opening: Context / pleasantries / reference to previous communication
  - Middle: The request / proposal / information
  - Closing: Expression of thanks, anticipation of response, offer of further assistance

**Move 4: Closing**
- "Hormat kami," = "Sincerely," / "Yours faithfully,"
- "Demikianlah" can precede closing for emphasis (more formal)

### 1.3 Micro-Level Patterns

**Reduplication for plurals and variety:**
- "barang-barang" = goods (plural)
- "harga-harga" = prices (various)
- "ketentuan-ketentuan" = provisions (all applicable provisions)

**NPWP format:**
`XX.XXX.XXX.X-XXX.XXX` — always group this way in translation, or convert to standard alphanumeric.

**Address format:**
- Street → RT/RW → Kelurahan/Desa → Kecamatan → Kota/Kabupaten → Provinsi → Kode Pos
- In English: reverse smallest-to-largest: Street → Sub-district → District → City → Province → Postal Code

## 3. Standards

### 3.1 Currency and Numeric Standards

- **Official currency:** Rupiah (IDR)
- **Symbol:** Rp (placed before amount, no space: Rp10.000,00 or Rp 10.000,00 — both seen)
- **ISO 4217 code:** IDR
- **Subunit:** Sen (1/100), but coinage practically obsolete; amounts rounded to nearest Rupiah in most transactions
- **Translation format:** Rp 1.000.000,00 → IDR 1,000,000.00 (formal) or IDR 1,000,000 (rounded)

**Decision rule for currency:**
- In financial/accounting contexts: Use "IDR" (ISO code)
- In general commercial contexts: "IDR" or "Rp" may both be acceptable
- Always convert decimal/thousand separators
- Never apply dollar/cent conversion — IDR is not pegged to USD

### 3.3 Business Licensing System (OSS)

Indonesia uses the **Online Single Submission (OSS)** system:
- **NIB (Nomor Induk Berusaha):** Business Registration Number — the primary identifier
- **Izin Usaha:** Business License
- **Izin Komersial atau Operasional:** Commercial/Operational License

### 3.4 Invoice Format Standards (Peraturan Pemerintah + Tax Office)

A standard Indonesian tax invoice (Faktur Pajak) must contain:
1. Nama, alamat, NPWP Penjual
2. Nama, alamat, NPWP Pembeli
3. Jenis barang/jasa
4. Harga jual/penggantian
5. PPN terutang
6. PPnBM terutang (if applicable)
7. Kode, nomor seri, dan tanggal faktur pajak

### 3.5 Weights and Measures Standards

Indonesia uses the metric system per:
- UU No. 2 Tahun 1981 tentang Metrologi Legal
- Commercial weights measured in grams, kilograms, tons
- Liquids in liters, milliliters
- Length in meters, centimeters, millimeters
- Temperature in degrees Celsius

### 3.6 Business Correspondence Etiquette

Key principles:
- **Tingkat tutur (speech level):** Use formal language until relationship is established
- **Gelar dan jabatan (titles):** Always preserve titles (Ir., Dr., Drs., S.E., M.M.). Do not translate degree abbreviations — they follow Indonesian naming conventions
- **Kami vs. Saya:** "Kami" (we) used for company voice. "Saya" (I) only for personal correspondence. In English, use "we" for company correspondence.
- **Pembuka lembut (soft opening):** Indonesian tends to open with pleasantries or reference to prior communication before stating the main point. Preserve this order — do not front-load the main point in translation unless the target document style demands directness (and the client has agreed).
