## 通用原则

- 使用正式书面语
- 法律术语精确，避免口语化表达
- 保持法律论证的逻辑结构（鉴于、因此、前提是）
- 精确保留条款编号（第1条、第2.3款）

## 条款表达

- 义务性条款使用"应当"（对应 shall）
- 授权性条款使用"可以"（对应 may）
- 禁止性条款使用"不得"（对应 shall not）

## 签署栏

- "Fait à Paris, le ..." → "签署于巴黎，……"
- "Le Gérant" → "管理人"
- "Pour la société" → "代表公司"
- 签名行：保持格式，不翻译人名

## 日期条款

- 精确保留所有日期
- "à compter du" → "自……起"
- "pour une durée de" → "期限为"

## 法律模糊性处理

### 一词多义
法律术语常有多个含义，需根据上下文判断：
- "acte" → 文件 / 行为 / 证书（取决于上下文）
- "engagement" → 承诺 / 义务 / 债务
- "titre" → 所有权 / 证券 / 标题
- "droit" → 权利 / 法律 / 费用

处理策略：
1. 分析条款的法律意图
2. 参考同文档中其他用法保持一致
3. 如无法确定，保留原文 + 注释可能的含义

### 管辖权差异
不同法律体系的同一概念可能不完全对等：
- 法国 "SARL" ≠ 美国 "LLC"（近似但治理结构不同）
- 中国 "有限责任公司" ≠ 法国 "SARL"（管理层构成有差异）

处理策略：
1. 使用目标语言中最接近的对等术语
2. 首次出现时加注说明差异（如 "SARL（类似于有限责任公司）"）
3. 不要自行创造法律概念

### 双语合同格式
- 保留两种语言的条款编号对应
- "以中文为准" / "la version française fait foi" → "以中文版本为准"
- 定义条款中的术语必须在两种语言中严格对应
- 交叉引用必须匹配目标语言的条款编号

## 读者模型

**谁在阅读中文法律译文，他们期望什么？**

法律文书的读者包括律师、法官、监管机构、合同管理人员以及协议各方当事人。他们阅读文书以确定权利、义务和法律后果。其核心期望如下：

- **法律意图的精确性** — 每一个用词都可能在诉讼中被严格审查。翻译者必须精准保留法律意图，不得引入解释性空白。
- **始终一致的正式语体** — 法律中文遵循既定的文书惯例和术语体系。口语化或非正式表达会损害文书的可执行性。
- **法律结构的完整保留** — 鉴于条款、正文条款、定义条款、签署栏和附件各有不同功能，其结构必须严格维持。
- **定义术语的一致性** — 定义术语每次出现时必须使用完全相同的译法。不一致的表述可能无意中产生新的含义。
- **法律体系的辨识意识** — 某一法律体系中存在的概念在另一法律体系中可能没有直接对应。译者必须在不歪曲的前提下进行桥接。
- **管辖权意识** — 不同司法管辖区的法律概念内涵不同。译者应识别概念所属的法律体系，并选择目标法律体系中最接近的对等表达。

**什么会破坏读者的信任？**
- 误译义务条款，导致风险或责任的归属发生改变。
- 创造目标司法管辖区中不存在的法律概念。
- 定义术语前后不一致，产生歧义。
- 日期、条款编号或签署栏中的错误可能导致条款无效。
- 擅自更改条款编号，破坏整份文书的交叉引用体系。

## 决策框架

### 1. 义务语言（Shall/May/Must 对应中文）

| 原文意图 | 中文处理 | 说明 |
|---|---|---|
| 约束性义务 | "应当" | 对应 shall，表示主体有义务执行 |
| 禁止性义务 | "不得" | 对应 shall not，表示主体被禁止执行 |
| 最高约束力 | "必须" | 对应 must，语气强于"应当"，常用于法规强制性要求 |
| 权限或自由裁量 | "可以" | 对应 may，表示允许但非强制 |
| 事实陈述或声明 | 使用陈述句，不用助动词 | 对应一般现在时，无需情态动词 |
| 附条件义务 | "在……前提下" | 对应 subject to 或 provided that，引出前提条件 |
| 承诺性义务 | "承诺" / "保证" | 对应 undertake 或 warrant，根据语境选择 |

### 2. 大陆法系与普通法系概念对照

| 原文概念 | 中文处理 | 说明 |
|---|---|---|
| SARL（法国） | 有限责任公司 | 首次出现时加注：SARL（有限责任公司） |
| SA（法国） | 股份有限公司 | 首次出现时加注：SA（股份有限公司） |
| SAS（法国） | 简化股份有限公司 | 法国特有形式，首次加注说明 |
| notaire（法国） | 公证人 | 大陆法系公证人职能宽于普通法系，首次加注 |
| huissier（法国） | 执达官 / 司法执达员 | 法国特有司法职务，需加注说明 |
| gérant | 管理人 | 对应大陆法系公司治理中的管理角色 |
| procuration | 授权委托书 | 不可与普通法系 power of attorney 完全等同 |
| charge hypothécaire | 抵押权负担 | 大陆法系物权概念，需准确对应 |
| prescription | 时效 | 注意与普通法系 limitation 期的细微差别 |

### 3. 合同结构（鉴于条款、正文条款、签署栏）

| 原文结构 | 中文处理 | 说明 |
|---|---|---|
| 鉴于条款（Whereas / Considérant） | 使用"鉴于"开头 | 每段以"鉴于"起首，保留背景事实陈述功能 |
| 正文条款（Clauses opératoires） | 使用"第X条"编号 | 保留原文条款编号体系，不重新编排 |
| 定义条款（Définitions） | 使用"定义"标题 | 术语定义必须与正文中出现的译法严格一致 |
| 陈述与保证 | "陈述与保证" | 对应 representations and warranties |
| 赔偿条款 | "赔偿" | 对应 indemnification，保留责任分配逻辑 |
| 终止条款 | "终止" | 对应 termination，保留终止条件的完整性 |
| 适用法律与管辖权 | "适用法律与管辖" | 对应 governing law and jurisdiction |
| 签署栏 | 见签署栏格式表 | 保留格式，不翻译人名 |

### 4. 日期格式转换规则

| 原文意图 | 中文处理 | 说明 |
|---|---|---|
| 法语日期（数字格式） | 转换为中文格式："2025年4月30日" | 消除日/月歧义，统一使用年/月/日顺序 |
| 日期含时间引用 | 保留时间精确表述 | 使用24小时制与文书其余部分保持一致 |
| 期间起始日期 | "自……年……月……日起" | 对应 à compter du，明确起算时点 |
| 期间终止日期 | "至……年……月……日止" | 保留终止时点的明确性 |
| 期间长度 | "期限为……" | 对应 pour une durée de，后接具体数字和单位 |

## 错误模式库

### 严重错误（零容忍）

| 错误类型 | 说明与指导 |
|---|---|
| 过度翻译实体类型 | 将外国法律实体缩写替换为国内对等实体，改变了实体的法律性质。应保留原文缩写并在首次出现时以括号加注说明 |
| and/or 处理不当 | 省略"/"会改变逻辑范围。当原文意图同时涵盖选择性和并列性时，必须保留"和/或"结构 |
| 日期格式歧义 | "04/05/2025"在不同司法管辖区含义不同。法律文书中应使用中文明确格式"2025年4月5日" |
| 定义术语不一致 | 同一定义术语出现两种不同译法，产生两种潜在含义。每次出现必须完全一致 |
| 条款编号变更 | 交叉引用依赖精确的条款编号。重新编号会破坏整份文书的引用体系 |

### 常见错误（实践中频繁出现）

| 错误类型 | 说明与指导 |
|---|---|
| 语体不一致 | 法律文书要求一致的正式书面语体。避免混用"应当"与口语化的"要"、"需要"、"应该"等表达 |
| "subject to" 位置错误 | "在……前提下"应置于被限制条款的开头，而非中间或末尾，以正确表达条款间的从属关系 |
| 签署栏格式变更 | 保留签名行、日期行和见证人行的精确布局。格式是法律效力的组成部分 |
| 术语搭配不当 | 法律术语有固定搭配，不可随意组合。如"违约责任"不可改为"违反合同的责任" |

### 隐蔽错误（容易遗漏）

| 错误类型 | 说明与指导 |
|---|---|
| 跨法律体系假对等 | 表面相同的术语在不同法律体系中可能产生不同的法律后果。必须在目标法律体系中核实该概念的实际含义 |
| 首次出现外来概念未加注释 | 当术语在目标语言中没有直接对等词时，首次出现必须包含解释性说明。后续出现可单独使用该术语 |
| 单复数定义术语 | 法律文书中定义术语通常使用单数形式。擅自改为复数可能无意中扩大或缩小定义范围 |
| 法律行为与法律事实混淆 | 区分"法律行为"（acte juridique）与"法律事实"（fait juridique），两者在中文中可能都被译为"行为"但法律意义不同 |

## 参考表

### 签署栏格式（中文惯例）

| 要素 | 中文惯例 | 说明 |
|---|---|---|
| 签署人姓名 | 保留原始拼写，不翻译 | 个人姓名使用原文形式 |
| 职务 | 翻译为中文职务名称 | 如"法定代表人"、"总经理"、"管理人" |
| 签名行 | 使用横线或"签字：" | 保留签名区域的格式 |
| 日期行 | "日期：" + 中文格式日期 | 使用年/月/日格式，如"2025年4月30日" |
| 见证人行 | "见证人：" + 姓名和签名行 | 保留见证人签名区域 |
| 公司代表 | "代表[公司名称]" | 对应 pour la société / for and on behalf of |
| 印章区域 | 标注"（盖章）"或留出印章位置 | 中国法律文书通常需要公章 |

### 法律文件数字格式

| 元素 | 中文格式 | 说明 |
|---|---|---|
| 金额 | 中文大写数字 + 阿拉伯数字（括号） | 如"伍万元整（¥50,000.00）" |
| 百分比 | 中文数字 + 百分号 | 如"百分之五（5%）" |
| 时间段 | 数字 + 单位 | 如"三十（30）个工作日" |
| 大额数字 | 使用标准分节符 | 如"壹佰万元（¥1,000,000.00）" |
| 日期数字 | 使用中文年/月/日格式 | 如"2025年4月30日"，避免纯数字格式 |
