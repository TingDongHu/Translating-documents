- 数值必须完全一致，零容错

## 读者模型

**谁在阅读这些财务文件？他们期望什么？**

财务文件的读者包括审计师、会计师、税务机关、投资者和财务分析师，他们依赖这些文件进行报告、合规和决策。他们期望：

- **每个数字都精确无误** — 财务报表是具有法律效力的证明文件。哪怕一个数字的差错，也可能歪曲一个组织的财务状况。
- **会计准则术语一致** — 术语必须与国际会计准则（IFRS）、中国会计准则（CAS）或法国会计准则（PCG）保持一致。读者期望能将术语映射到标准会计概念。
- **数值格式清晰无歧义** — 小数点、千分符和货币符号必须遵循中文目标市场的格式规范，且不得改变原值。
- **审计就绪的结构** — 表格、脚注、交叉引用和辅助明细表必须完整保留，使每一个数字都可以追溯和验证。

**什么会破坏他们的信任？**

- 数值与原文不符 — 即使只是四舍五入的误差，也会使整份文件失效。
- 货币符号指向了非原文所指的货币。
- 财务报表行项目标签翻译错误，导致分析偏差。
- 日期范围与报告期间不匹配。

## 决策框架

### 1. 数值格式

| 原文格式 | 中文处理 | 说明 |
|---|---|---|
| 逗号作小数分隔符（如 100,00） | 转换为句点：100.00 | 法语系国家常用逗号作小数分隔符 |
| 句点作千分符（如 1.234） | 转换为逗号：1,234 | 法语中句点用于千分位 |
| 空格作千分符（如 1 234） | 转换为逗号：1,234 | 法语/德语常见格式 |
| 无千分符（如 1234567） | 添加千分符：1,234,567 | 提升中文财务文件可读性 |
| 百分比使用逗号小数（如 15,5%） | 转换为句点小数：15.5% | 中文统一使用句点作小数分隔符 |

### 2. 货币处理

| 场景 | 中文处理 | 说明 |
|---|---|---|
| 货币代码出现（如 EUR、USD） | 保留三字母 ISO 代码不变 | 确保国际通用性 |
| 货币符号出现（如 €、$） | 正式文件中首次使用时加注 ISO 代码 | 如：欧元（EUR）1,000.00 |
| 原文以外币计价 | 保留原币种和原金额，不擅自换算 | 未经客户授权不得进行货币换算 |
| 文字金额与数字金额并列 | 两种形式均保留，并确保一致 | 如：壹万元整（10,000.00 元） |
| 文中出现汇率 | 保留汇率数值，翻译标签 | 如："买入价"、"卖出价" |

### 3. 财务报表术语

| 场景 | 中文处理 | 说明 |
|---|---|---|
| 标准 IFRS/CAS 概念 | 使用中文会计准则标准术语 | 如 bilan → 资产负债表 |
| 原文使用非标准标签 | 采用标准中文对应行项目名称 | 如有实质差异，加译注说明 |
| 小计或合计行 | 保留层级结构，术语一致 | 如"小计"、"合计"、"总计" |
| 附注引用 | 保留上标编号或符号 | 确保脚注编号与正文一致 |

### 4. 报告期间日期

| 场景 | 中文处理 | 说明 |
|---|---|---|
| 单一日期 | 使用明确格式：2025年12月31日 | 避免使用易混淆的短格式 |
| 会计期间日期范围 | 使用"截至2025年12月31日止年度" | 中文财务报告标准表述 |
| 财年引用 | 使用"2025财年"，全文保持一致 | 中文语境下避免使用 FY2025 |

## 错误模式库

### 严重错误（零容忍）

| 错误类型 | 中文处理指导 |
|---|---|
| **负值格式错误** | 法语中负值有时用括号表示如 (1 234,56)，中文财务报告中通常使用负号如 -1,234.56。必须按中文财务规范处理，不得遗漏负号 |
| **四舍五入导致数值变动** | 任何数字的四舍五入都可能改变财务报表中的关键数据。原文为 1,234.567 时，必须保持原精度，不得舍入为 1,234.57 |
| **零值与短横线混淆** | 财务表格中"0.00"表示零值，"—"或短横线表示"不适用"。二者含义不同，不可互换 |
| **脚注引用丢失** | 脚注标记与正文数字披露直接关联，丢失会导致审计追溯链断裂。必须完整保留所有上标编号 |

### 常见错误

| 错误类型 | 中文处理指导 |
|---|---|
| **货币符号位置不当** | 中文规范中货币符号通常置于数字前如 ¥1,234.56 或在正式文件中使用"人民币1,234.56元"。避免使用美元符号置于数字后的格式 |
| **小数位数不一致** | 同一文件中数值精度必须统一。资产负债表通常保留两位小数，利润率保留一位。不得在同一表格中混用不同精度 |
| **日期格式混淆** | 同一文件中不得混用"2025/01/01"与"2025年1月1日"。中文财务文件应统一使用"YYYY年MM月DD日"格式 |

### 隐蔽错误

| 错误类型 | 中文处理指导 |
|---|---|
| **会计准则术语假对等** | 法国 PCG 中的"provisions"与中文"预计负债"并非完全等同。翻译时需确认具体语境，避免因术语假对等导致会计概念偏差 |
| **审计意见翻译偏差** | "réserve"在审计语境中应译为"保留意见"而非字面意义的"准备金"。审计报告术语有严格的专业对应关系，不可随意翻译 |
| **非标准简称误用** | 法语财务文件中常用的缩写如"TTC"（含税）、"HT"（不含税）必须完整翻译为中文全称，不可保留缩写 |

## 参考表

### 财务报表标准结构（6 个要素）

| 要素 | 说明 | 常见行项目 |
|---|---|---|
| 资产负债表 | 反映特定时点的财务状况 | 资产、负债、所有者权益 |
| 利润表 | 反映一定期间的经营成果 | 营业收入、营业成本、净利润 |
| 现金流量表 | 反映一定期间的现金流入流出 | 经营活动、投资活动、筹资活动 |
| 所有者权益变动表 | 反映所有者权益的增减变动 | 实收资本、资本公积、盈余公积 |
| 财务报表附注 | 对报表项目的补充说明 | 会计政策、估计变更、或有事项 |
| 审计报告 | 独立审计师的意见 | 无保留意见、保留意见、否定意见 |
