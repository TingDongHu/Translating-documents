## 通用原则

- 适应中文商务表达习惯
- 营销内容应本地化，而非直译
- 保持品牌名和商标不变
- 计量单位和货币按中文惯例转换

## 商务信函

- 使用中文商务信函格式
- "Madame, Monsieur" → "尊敬的先生/女士"
- "Cordialement" → "此致敬礼"

## 营销和提案

- 适应中文表达习惯，不要直译
- 保持说服力，同时符合中文文化

## 广告文案翻译

### 核心原则
广告翻译 = 本地化（transcreation），不是直译。
目标：在目标市场产生与原文等效的情感冲击和说服力。

### 常见场景
1. **产品功效描述**：夸张修辞需要保留力度，但符合目标市场广告法规
2. **用户评价/口碑**：口语化、情感化，保留真实感，不要正式化
3. **促销信息**：数字、折扣必须精确，文案风格可以本地化
4. **品牌故事**：叙事风格适应目标文化，核心价值观不变

### 品牌名翻译策略
- 进入中国市场：音译选吉利字（Benz → 奔驰）、意译（Volkswagen → 大众）、音意兼顾（BMW → 宝马）
- 进入英语市场：保留原文或简化拼写
- 品牌 slogan：重新创作，不直译

### 源文谐音梗处理
如果源文含有谐音梗（中文广告常见）：
1. 还原本意（不是字面意思）
2. 品牌名保留音译
3. 广告语用目标语言的修辞手法重新创作
4. 参见 rules/base.md "创意语言处理"

## 商业合同条款

### 模糊条款处理
- "合理的"（raisonnable）→ "合理的" — 保留模糊性，这是法律意图
- "包括但不限于"（y compris mais sans s'y limiter）→ "包括但不限于"
- "在合理范围内"（dans la mesure du possible）→ "在合理可行的范围内"
- "在……前提下"（sous réserve de）→ "在……前提下"
- "尽管"（nonobstant）→ "尽管"

### 价格/付款条款
- 净价/毛价：HT → 不含税 / TTC → 含税
- 付款期限：à 30 jours → 30天内付款 / à réception → 收到即付
- 折扣：escompte de 2% → 2% 提前付款折扣 / remise → 折扣
- 违约金：pénalités de retard → 逾期付款违约金

### 质保条款
- "garantie" → "质保" / "保修"
- "garantie légale" → "法定质保"
- "garantie commerciale" → "商业质保"
- "vice caché" → "隐蔽瑕疵"

## 电商/数字营销

### 产品描述
- 特性列表：保持简洁、有力、可扫描
- SEO 关键词：在目标语言中寻找等效搜索热词
- 用户评价：保留口语化风格，不正式化

### 平台特定语言
- 产品标题：遵循目标平台的字数限制和格式规范
- 卖点列表：以利益点开头，而非功能点
- 行动号召：适配目标市场习惯（"立即购买" vs "Shop now"）

## 读者模型

**谁会阅读这些文档，他们有什么期待？**

商业文书的读者包括企业高管、采购负责人、销售经理和法务代表，他们基于文档内容做出商业决策。他们期待：

- **义务明确** — 谁必须在什么时间之前完成什么事项、以什么成本。承诺中的模糊表述会导致纠纷。
- **日期精确** — 交货日期、付款期限和有效期必须准确无误，不得产生歧义。
- **财务清晰** — 单价、折扣、罚金和币种在全文中必须一致且不可误解。
- **语气专业** — 直接、简洁、得体。商业读者不会容忍浮夸或含糊的语言。
- **品牌完整** — 营销材料必须保持品牌调性和说服力，同时符合目标文化。

**哪些问题会破坏读者的信任？**
- 贸易术语翻译错误，导致买卖双方的风险分配发生变化。
- 价格或数量前后不一致，暗示存在计算错误。
- 交货日期缺失或表述模糊。
- 品牌名称或口号翻译后削弱了品牌辨识度。

## 决策框架

### 1. 义务语言层级

| 源文模式 | 中文处理 |
|---|---|
| 强义务（对应法语"doit/est tenu de"） | 使用"应"/"须" — 建立具有法律约束力的合同义务 |
| 中等义务（对应法语"doit"外部要求） | 使用"应"/"须" — 适用于法律法规、标准规范施加的义务 |
| 推荐性（对应法语"devrait"） | 使用"应"/"建议" — 非强制性，属于建议性质 |
| 许可（对应法语"peut"） | 使用"可"/"有权" — 授予选择权，而非施加义务 |
| 禁止（对应法语"ne doit pas"） | 使用"不得"/"禁止" — 根据义务强度选择 |

### 2. 公司法律形式

| 源文实体类型 | 中文对应处理 |
|---|---|
| SARL（有限责任公司） | 保留原文缩写，首次出现时括注"有限责任公司" |
| SA（股份有限公司） | 保留原文缩写，首次出现时括注"股份有限公司" |
| SAS（简易股份有限公司） | 保留原文缩写，首次出现时括注"简易股份有限公司" |
| SNC（普通合伙企业） | 保留原文缩写，首次出现时括注"普通合伙企业" |
| 无直接中文对应的形式 | 保留原文缩写，括号中注明中文释义（如"AB（股份公司）"） |

### 3. 商业数量表达

| 场景 | 中文处理规则 |
|---|---|
| 以原始单位表达的数量 | 保留原始数量。仅在目标市场商业惯例使用不同单位且客户授权转换时，才进行单位换算 |
| 包装数量（打、令、罗） | 保留原始数值，使用中文商业等效单位 |
| 基于体积的定价（每吨、每立方米） | 保留单位。如客户要求，可在括号中添加目标市场等效单位 |

### 4. 国际贸易术语（Incoterms）

| 场景 | 中文处理规则 |
|---|---|
| 源文中出现贸易术语 | 保留英文缩写（如 FOB、CIF、EXW）。贸易术语为国际标准，不得翻译 |
| 贸易术语附带地点 | 保留原始地点名称。不得翻译港口或地点名 |
| 贸易术语含版本年份 | 保留年份标注（如"Incoterms 2020"） |

## 错误模式库

### 严重错误（零容忍）

| 错误类型 | 危害说明 |
|---|---|
| **日期歧义** | 将日期格式混用导致歧义：DD/MM/YYYY 与 MM/DD/YYYY 可能被误读。必须使用明确格式"2025年4月30日"或"2025-04-30" |
| **货币混淆** | 使用货币符号（$）而非三字母代码（USD）。$ 可能是美元、加元或澳元，造成金额重大错误 |
| **小数/千分符错误** | 中文环境下"1.500"可能被误读为"1,500"或"1.5"。必须明确标注小数点和千分符，或使用文字表述 |

### 常见错误（实践高频）

| 错误类型 | 纠正指南 |
|---|---|
| **Incoterms 翻译错误** | 贸易术语不得意译。FOB 不应翻译为"船上交货"，应保留"FOB"。仅在首次出现时可在括号中注明中文释义 |
| **付款条款格式不统一** | 付款条件格式必须全文一致。"30天账期"与"付款期限30日"不可混用，选择一种并贯穿全文 |
| **义务语言混用** | "应"/"须"/"可"不得互换。合同义务使用"应"，法律要求使用"须"，许可使用"可" |

### 隐蔽错误（易被忽视）

| 错误类型 | 纠正指南 |
|---|---|
| **品牌语气偏差** | 营销材料翻译后语气与原文不一致。需核对：原文是正式还是活泼？翻译后是否保持了同等的说服力和情感调性？ |
| **营销文案直译** | 广告语、宣传文案逐字翻译会导致表达生硬。必须进行创意本地化，而非直译，同时保持核心卖点 |
| **商务信函语气错位** | 中文商务信函应保持一致的正式程度。不应在同一篇函件中混用"尊敬的客户"和过于口语化的表达 |
