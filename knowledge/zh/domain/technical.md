## 通用原则

- 使用行业标准术语（ISO、GB 等）
- 精确保留所有测量值、公差和规格
- 使用国际单位制；仅在明确要求时转换
- 全文术语保持一致

## 技术写作风格

- 操作说明使用祈使句（"按下按钮"而非"您应该按下按钮"）
- 规格说明使用被动语态
- 简洁明了，避免冗余
- 有序步骤使用编号列表

## 单位和测量

- 保留原始单位；如需要可加注国际单位制
- 温度：保留摄氏度，如需要加注华氏度
- 压力：保留原始单位（bar、Pa、psi）

## 标准和认证

- 精确保留标准编号（ISO 9001、CE 标志等）
- "marquage CE" → "CE 标志"
- "certification ISO" → "ISO 认证"

## 读者模型

**技术文书的读者是谁？他们有什么期待？**

技术文书的读者包括工程师、技术员、操作人员和维修人员，他们查阅文书的目的是正确、安全地执行任务。他们期待：

- **精度至上** —— 每一个测量值、公差和规格都必须与原文完全一致。一个错误的数值就可能导致设备损坏、安全事故或产品失效。
- **术语一致** —— 如果某零部件在第 5 页被称为"执行器"，就不应在第 20 页变成"驱动器"。读者依赖术语的稳定性来快速检索信息。
- **步骤清晰** —— 操作说明的每一步都必须毫无歧义。读者不应需要猜测下一步动作，也不应需要自行推断关键的安全预防措施。
- **遵循标准惯例** —— 单位、符号、缩写和格式应遵循公认的工程标准（ISO、IEC、ANSI）。任何偏离都会削弱读者的信任。

**什么会破坏他们的信任？**
- 型号或零件号翻译错误，导致采购了错误的零部件。
- 安全警告等级不正确（例如：需要"危险"的地方标为"注意"）。
- 测量值未转换或转换错误，导致装配不合格。
- 交叉引用不一致，将读者引导到错误的章节。

## 决策框架

### 1. 单位和测量

| 情况 | 决策 |
|---|---|
| 源文包含带单位的测量值 | 保留原始数值；将单位格式转换为中文公制惯例（数值与单位之间加空格）。 |
| 源文使用非法语单位（法语原文） | 保留原始单位和数值。除非项目说明书明确要求转换，否则不得转换为公制。 |
| 源文混用计量单位 | 完全保留混用状态，不要统一化。 |
| 温度以摄氏度给出 | 保留摄氏度。仅在项目说明书要求时加注华氏度。 |
| 出现组合单位（如 N-m、kg/m³） | 保留单位符号原样。遵循 ISO 80000 标准工程记法。 |

### 2. 安全警告（ANSI Z535 信号词中文映射）

| 情况 | 决策 |
|---|---|
| 信号词表示可能导致死亡或严重伤害的即时危险 | 映射为 **危险**（ANSI Z535：白底红字）。 |
| 信号词表示可能导致严重伤害的潜在危险 | 映射为 **警告**（ANSI Z535：黑底橙字）。 |
| 信号词表示轻微或中度伤害风险或财产损失 | 映射为 **注意**（ANSI Z535：黑底黄字）。 |
| 源文包含安全符号或象形图 | 保留符号引用原样。仅翻译伴随的文字。 |

### 3. 交叉引用

| 情况 | 决策 |
|---|---|
| 按编号引用其他章节 | 保留章节编号原样。 |
| 按名称引用其他章节 | 使用统一术语翻译章节名称。 |
| 引用图表或示意图 | 保留图表编号原样。翻译标题（如有）。 |
| 页码引用 | 保留原始页码（若 DTP 排版重排后，使用目标布局页码）。 |

### 4. UI 标签和控制指示器

| 情况 | 决策 |
|---|---|
| 物理按钮、开关或指示器上的标签 | 翻译为标准中文 UI 术语（如"开/关"、"停止"、"复位"）。 |
| 文档中嵌入的软件界面字符串 | 翻译字符串；若软件本身未本地化，用方括号注明。 |
| 标签包含键盘快捷键 | 保留快捷键组合原样。翻译操作描述。 |

## 错误模式库

### 严重错误（零容忍）

| 错误类型 | 说明 |
|---|---|
| **假对等术语** | 源文中某些术语看似与中文术语对应，实则含义不同。必须逐条核对领域术语表，不可凭直觉翻译。 |
| **小数逗号遗漏** | 法语使用小数逗号（如 3,14），中文必须使用小数点（3.14）。遗漏会导致数值含义改变。 |
| **千分符错误** | 源文中的千分位分隔符（空格或句点）在中文输出中必须使用逗号或省略，不可直接保留。 |
| **日期格式错误** | 技术文书中的日期（如修订历史）必须使用明确格式："2025-04-30"或"2025年4月30日"，绝不可出现"04/05/2025"等歧义格式。 |
| **安全警告降级** | 将"危险"误译为"警告"或"注意"会破坏危险等级传达体系，可能导致法律责任。 |

### 常见错误（实践中频繁出现）

| 错误类型 | 指导 |
|---|---|
| **单位符号错误** | 严格遵循 ISO 80000 单位格式。数值与单位之间加空格（如"100 mm"而非"100mm"）。 |
| **技术缩写未展开** | 工业缩写（PLC、HMI、CNC、PID）和标准缩写（max、min、avg、ref）为国际通用，保留原样。首次出现时可加注中文全称。 |
| **图表编号错位** | 翻译后图表编号必须与正文引用完全对应。若原文为"图 3"，翻译后不可变为"图 4"。 |

### 隐蔽错误（容易遗漏）

| 错误类型 | 指导 |
|---|---|
| **标准编号格式错误** | 标准编号（如 ISO 9001:2015）必须保留原样，不可修改编号或年份。 |
| **CE/UL 认证术语混淆** | "marquage CE"（CE 标志）与 UL 认证是不同体系，不可混用。"CE"在中文中直接保留为"CE"。 |
