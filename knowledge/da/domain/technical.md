# Technical Domain: Danish Translation Standards

## Reader Model

**Primary Reader:** Danish software developer (softwareudvikler), system administrator (systemadministrator), or IT project manager (IT-projektleder). The reader expects precise technical terminology, consistent use of Danish IT terms where they exist, and natural handling of English loanwords that are fully integrated into Danish IT language. They are comfortable with English technical terms but prefer Danish equivalents when they are established and natural.

**Secondary Reader:** End user reading Danish software interface, documentation, or technical guide. They expect clear, accessible Danish that avoids unnecessary jargon while maintaining technical accuracy.

**Tertiary Reader:** Technical writer or localisation engineer reviewing Danish technical translation for consistency and quality. They expect adherence to established Danish IT terminology standards.

**Reader Priorities:**
1. Consistent terminology (same term for same concept throughout)
2. Natural handling of English loanwords (Danish accepts many English IT terms)
3. Clear, concise instructions (especially in UI and documentation)
4. Correct technical formatting (code blocks, variables, commands preserved)
5. Appropriate register (technical but accessible)
6. Consistent use of Danish equivalents for common IT concepts

## Error Pattern Library

### Common Translation Errors

| Error Type | Incorrect | Correct | Explanation |
|-----------|-----------|---------|-------------|
| English loanword | Unnecessary Danish coinage | Established loanword | Use the term Danish users actually use |
| UI string | "Klik på linket" (too formal) | "Klik på linket" (correct) | Verify consistency; "link" is standard |
| Variable | Translating code variables | Preserve original | NEVER translate code, variables, or commands |
| Technical term | "lager" (warehouse) for "storage" | "lagring" (data storage) | Context-dependent: "lager" = physical; "lagring" = digital |
| Error message | "En fejl er opstået" (passive) | "Der opstod en fejl" / "Fejl" | Concise error messages preferred |
| Verb form | "Du skal trykke på..." | "Klik på..." / "Tryk på..." | Use standard Danish action verbs in UI |
| Plural | "en email" / "to emailer" | "en e-mail" / "to e-mails" | "E-mail" plural: "e-mails" (common) |
| Capitalisation | Capitalising common nouns | Lowercase for common nouns | Danish does not capitalise common nouns |

### Ambiguous Terms Requiring Context

| English | Danish Options | When to Use |
|---------|---------------|-------------|
| "run" | "køre" (execute) / "afvikle" (process) | "køre" for programs; "afvikle" for processes |
| "handle" | "håndtere" (process) / "behandle" (treat/process) | "håndtere" for data/tasks; "behandle" for cases |
| "address" | "adresse" (location) / "adresse" (network) | Same word; context determines meaning |
| "driver" | "driver" (device) / " chauffør" (person) | "driver" for device driver; "chauffør" for person |
| "port" | "port" (network) / " havn" (maritime) | "port" for network port; "havn" for maritime |
| "service" | "tjeneste" (digital) / "service" (maintenance) | "tjeneste" for web services; "service" for maintenance |

### Danish Software Localisation Conventions

1. **Date format in UI**: DD.MM.YYYY or DD. MMM YYYY (e.g., "15. jan. 2026")
2. **Time format in UI**: HH:MM (24-hour, e.g., "14:30")
3. **Number format in UI**: 1.000,00 (dot thousands, comma decimal)
4. **Currency in UI**: 100,00 kr. (symbol after amount)
5. **First day of week**: Monday (mandag) — ISO standard
6. **Measurement units**: Metric system (meter, kilogram, liter, grader Celsius)
7. **Decimal separator**: Comma (,) in numbers and currency
8. **Thousands separator**: Dot (.) in numbers and currency
